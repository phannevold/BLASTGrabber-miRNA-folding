miRNA Folding User Manual

miRNA Folding is a plugin for BLASTGrabber, a BLAST result management and analysis tool developed by the Molecular Evolution Research Group (MERG) at the Department of Biology, University of Oslo. miRNA Folding not usable outside of the BLASTGrabber environment.

miRNA Folding adresses a very specific need for researchers working with microRNA (miRNA). BLAST searches will frequently return a large number of hits. When searching for short sequences of miRNA, usually 18-24 bases in length, the number of hits can be overwhelming. A number of these sequences may not be actual candidates for miRNA.

This plugin allows the user to fold precursor sequences of miRNA using the Vienna RNA tools RNAfold algorithm, to see if the segment forms a stable stem/hairpin loop structure, and thereby judge the validity of candidate hits.

Accessing the plugin

The plugin accepts data from BLASTGrabber through the latter’s Clipboard view. A dataset needs to be either opened or imported in BLASTGrabber, and viewed through the Sequence viewer, Matrix viewer or Description viewer. Select candidate sequences from either of these by marking them. Optionally several can be marked with Ctrl+left mouse button or Shift+left mouse button. Right click selected sequences and select Quick-grab selected sequences from the context menu.

From the Clipboard view, mark selected sequences (or Select all) and right-click again. Select Plugins > miRNA Folding from the context menu. The marked sequences are exported to miRNA Folding, and you are presented with the miRNA Folding main window.

Using the plugin

The top tree view lists the sequences that were imported to miRNA Folding. Each folder is a BLAST query, under which corresponding hits can be found. Buttons to the left of the tree view allow you to expand and collapse the tree, select queries only, hits only, or all sequences. Help is available at any time by pressing the Help button. The bottom-left window panel displays a graphical view of the folding stucture. The bottom-right panel contains several tabs with commands and data outputs.

Understanding and setting options

The Options tab gives a set of common options for the folding tools.

Scale to temperature. This option allows you to scale the results of the folding algorithm to a certain temperature, eg. for poikilothermic organisms. The default temperature is 37°C. As the folding algorithm is based on thermodynamics, setting the temperature can have a profound effect on the folding structure. See also the temperature range option in the Multiple runs tab.

Assume circular RNA. This option makes the folding algorithm assume RNA molecules are circular in structure, rather than linear.

Disallow lonely pairs (helices of length 1). When selecting this option, a single pair of bases with a pairing between them and bulges or loops on either sides, are disallowed.

 

The following set of options determine how dangling ends are treated:

Ignore dangling ends.

Only unpaired bases can participate in at most one dangling end.

Dangling energies will be added for the bases adjacent to a helix on both sides.

Allow coaxial stacking of adjacent helices in multi-loops.

 

The following set of options determine whether to allow GU pairs:

Allow GU pairs. Guanine-Uracil pairs are allowed.

Disallow GU pairs at ends of helices. Guanine-Uracil pairs are allowed, except at the ends of helices, where energy conditions are more likely to pull GU pairs apart.

Disallow GU pairs. Guanine-Uracil pairs are entirely disallowed.

 

Use parameter file. The Vienna RNA tools allow the use of parameter files, with thermodynamic parameters that influence the folding algorithm. A small number of parameter files are supplied in the par folder. Click the ‘...’ button to select and load a parameter file.

Folding a single sequence

Folding single sequences are done from the MFE structure, or Minimum Free Energy structure tab. Simply mark a sequence in the tree view, either a query or a hit, and press Fold. The results are displayed as a graphical representation of the secondary structure on the left, and resulting parameters on the right.

Folding suboptimal structures

The Suboptimal structures tab can be used to fold structures that deviate from the MFE structure. The first option gives structures that deviate from the MFE structure by a set in kcal/mol. The second option gives a random selection of n structures, drawn with probabilities equal to their Boltzmann weights via stochastic backtracking in the partition function.

Be aware that the number of suboptimal results increases exponentially with kcal/mol . A high setting (> 12-13 kcal/mol) will likely run you out of memory. Refer to the BLASTGrabber manual to allow for a larger heap space.

On pressing Fold suboptimal structures, the table is filled with results. Selecting a line in the table will render the secondary structure in the graphical view. Additional result data, as per a single sequence, can be found by switching to the MFE structure tab.

Folding multiple structures

The Multiple runs tab allows you to select multiple sequences from the tree view. Please note that a large number of sequences will take a while to fold. Warnings are given accordingly.

There is an option to fold a single sequence over a range of temperatures. With this option checked and a temperature range set, folding is repeated in 1° increments.

On pressing Fold selected sequences, the table is filled with results. Selecting a line in the table will render the secondary structure in the graphical view. Additional result data, as per a single sequence, can be found by switching to the MFE structure tab.

Understanding the folding output

The graphical view

This window shows you a graphic representation of the secondary structure. If Show pair probabilities is set, colors represent pair probabilities as given by the legend in the upper-right corner. For unpaired bases, colors represent the probability of being unpaired. If Show positional entropy is set, colors represent positional entropy as given by the legend. If an Accession code is found in a query name, and a match is found in the miRBase database, the mature miRNA segment is outlined in dashes. If the folded sequence is a BLAST hit, the BLAST alignment is given as the bases outlined in dots.

The parameter readout

The first and second lines of the parameter readout shows the name and base sequence of the folded sequence. The third line shows the structure in dot-parenthesis form and free energy value for the MFE structure. The fourth and fifth lines show the structure and free energy value of the thermodynamic ensemble and centroid structure, respectively. The sixth line shows additional parameters for the resulting thermodynamic ensemble. The free energy values are repeated below for clarity. If an Accession code is found in a query name, and a match is found in the miRBase database, the mature miRNA segment is shown. For hits, the alignment between query and hit is shown.
